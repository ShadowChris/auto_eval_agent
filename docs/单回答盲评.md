# 单回答盲评 · 功能文档

web 评估台（`python -m auto_eval.web.server` → http://localhost:8501）四个模式（单回答盲评 / 两回答对比 / 接模型在线评估 / 过程盲评）之一。对**用户提供的单个回答**做盲评打分，不涉及两个模型对比。

## 一、功能概述

- **盲评**：裁判全程看不到参考答案，独立判断回答好坏。
- **多裁判集成**：可勾选多个裁判，各自打分后聚合（多数投票 + 一致率检测）。
- **裁判 agent loop**：裁判可自主联网查证（`web_search` / `fetch_page` / `calculate` / `python_run`），核查事实/计算，不只凭印象打分。
- **主席仲裁**：多裁判分歧大时，主席裁判看全部理由后给出最终结论。
- **元评测（可选）**：填了参考答案的题，用客观真值（EM/F1/语义相似度）校验"裁判判得准不准"。
- **按垂域分析**：每题归一到垂域（Skill），跑完后给出分垂域的总览表。
- **实时 + 可导出**：SSE 逐题推送结果，跑完可导出 CSV/JSON。

## 二、输入

### 输入方式（二选一）
1. **粘贴文本**到输入框 →「解析预览」
2. **上传文件**（`.jsonl` / `.txt` / `.csv` / `.json`）→ 自动作为 JSONL 或文本解析

### 文本格式（每行一题，`|||` 分隔）
```
query ||| answer [||| reference]
```
- 至少需 `query ||| answer`，否则该行报错（计入"错误明细"）。
- 第三段 `reference` 可选，填了会跑元评测。

示例：
```
光合作用是什么？ ||| 植物利用光合成有机物 ||| 绿色植物利用光能将二氧化碳和水合成有机物并释放氧气
中国最长的河流？ ||| 长江
```

### JSONL 格式（每行一个 JSON 对象）
| 字段 | 必填 | 说明 |
|---|---|---|
| `question`（或 `query`） | ✅ | 题目 |
| `answer` | ✅ | 待评回答 |
| `reference` | ⬜ | 参考答案，填了跑元评测 |
| `category` | ⬜ | 垂域类目（如"事实"/"计算"）；不填则裁判自动分类 |

示例：
```jsonl
{"question":"中国最长的河流？","answer":"长江","reference":"长江","category":"事实"}
{"question":"写首春天的诗","answer":"春风又绿江南岸","category":"开放-创作"}
```

> 缺 `answer` 的行会报错跳过。解析后显示"预览（N 题）"表格，确认无误再开始评估。

## 三、评估配置

| 配置 | 说明 |
|---|---|
| **裁判** | 多选（来自 `config/judges.yaml`），每个带 persona；至少选一个 |

## 四、输出

### 1. 实时逐题结果（SSE）
每题评完即推送到结果表，无需等全部完成；进度条显示 `已完成/总数`。

### 2. 结果表格（每题一行）
| 列 | 含义 | 状态 |
|---|---|---|
| **题号** 🆕 | 题目 id（未提供时为 `q{序号}`），与按垂域问题分布里的题号对照、便于核查 | 待实施 |
| **垂域** 🆕 | 该题归一的垂域（分类后的 category） | 待实施 |
| 题目 / 回答 | 输入原文 | 已有 |
| **判定** | `right` / `wrong` / `partial` / `unclear`（无法确定时给 unclear，不硬猜） | 已有 |
| **总分** | 各维度加权总分（满分 5） | 已有 |
| **各维度** | 一级维度细分（准确性/完整性/相关性/有用性/安全性…，按垂域不同） | 已有 |
| **联网** | 裁判是否触发联网查证 | 已有 |
| **截断** | 是否因步数耗尽被强制判定 | 已有 |
| **仲裁** | 是否经主席仲裁（含置信度） | 已有 |
| **与真值** | 元评测对照：✓一致 / ✗不一致（仅有参考答案的题） | 已有 |
| **理由** | 裁判一句话总结 | 已有 |

部分题下方会展开 **🔍 裁判查证轨迹**（工具调用链），体现联网核查过程。

### 3. 总结区（全部完成后）
| 指标 | 含义 |
|---|---|
| 完成 `done/total` | 评完 / 总题数 |
| **平均总分** | `mean_total` / 归一化百分比 |
| **裁判判题准确率** | 有参考答案题里，盲判定对客观真值的比例（`meta_n` 题） |

### 4. 📊 按垂域分析（全部完成后）
每题经 `SkillRouter` 归一到垂域（未标 `category` 时裁判自动分类，带"通用"出口），按垂域聚合。**增强后**字段：

| 列 | 含义 | 状态 |
|---|---|---|
| 垂域 | 中文名（数学解题 / 音乐 / 影视 / 体育 / 新闻 / LBS（旅行规划） / 汽车 / 数码3C / 搜索 / 文档 / 通用） | 已有 |
| 样本量 | 该垂域题数 | 已有 |
| 正确率 | 该垂域 right 占比 | 已有 |
| 平均分 | 该垂域平均总分 | 已有 |
| **维度问题分布** 🆕 | 各一级维度（准确性/完整性/相关性/有用性/安全性…）的"问题占比"（维度分 ≤ 阈值的题占比），柱状图展示；点击某维度可下钻关联题号清单 | 待实施 |
| 首要错因 | 该垂域最大的错因簇（按 error_type 归一聚类），关联题号 | 已有(待加题号) |
| 低一致率 | 多裁判分歧占比（高则结论需人工复核） | 已有 |
| 分类失败占比 | 落到"通用"兜底的题占比（高 = 垂域覆盖有缺口） | 已有 |

> 原"类目覆盖"列**移除**（信息量低、与垂域高度重叠）。维度问题分布关联题号，可点开核查具体题，提升评估可信度。

### 5. 导出
- **CSV**（UTF-8-SIG，Excel 友好）：逐题扁平结果
- **JSON**：含 `task_id` / `mode` / `results`（逐题）/ `summary`（含 `by_skill`）

## 五、评估原理速览

```
输入(query + answer [+reference])
  → 解析为题目列表
  → 逐题盲评：裁判评分(可联网查证) → 多裁判集成 → 分歧则主席仲裁
  → [有 reference] 元评测：盲判 vs 客观真值 → 判题准确率
  → 垂域归一(SkillRouter) → 按垂域聚合 + 错因聚类
  → 实时输出逐题结果 + 总结 + 垂域分析 + 导出
```

**判定诚实原则**：无法确定对错时给 `unclear` 而非硬猜；`partial` 仅用于"方向对但不完整/有小错"。

## 七、按垂域分析增强方案（待确认后实施）

针对当前按垂域分析信息量不足的问题，增强方案如下。

### 7.1 维度问题分布（一级，必做）
**口径**：在某垂域的全部题里，各一级维度被判"低分（问题）"的题占比。
- 问题题定义：该维度分 ≤ 阈值（默认 ≤2，满分 5；阈值可配）
- 数据来源：每题 `verdict.rubric`（一级维度分），已具备，无需改裁判
- 输出结构：
  ```
  {维度: {"rate": 问题占比, "item_ids": [问题题号, ...]}}
  ```
- 示例：事实垂域 → 准确性问题 30% [q3, q7]，完整性问题 10% [q3]

**关联题号**（解决"难以信服可靠性"）：每个维度的问题分布带题号清单，前端可点击展开看具体题目（题号 + query + 该维度分），用户可逐题核查。

### 7.2 二级维度问题分布（可选，建议二期）
一级维度下的二级细分（如准确性 → 事实准确性 / 逻辑正确性 / 信息时效性）。
- **障碍**：当前 `verdict.rubric` 是一级扁平分（`_flatten_rubric` 已把二级平均成一级），二级分**未保留**。
- 要做需：裁判原始嵌套输出单独留存 → 涉及 schema / 聚合 / 展平逻辑改动，成本较高。
- 建议：一期只做一级，二级视一期效果再定。

### 7.3 可视化（饼图 / 柱状图）
引入图表库（CDN，无构建步骤）：
- **柱状图**：各垂域的维度问题分布（x=维度，y=问题占比）
- **饼图**：垂域样本量分布；或 correctness 分布（right/wrong/partial）
- 候选库：Chart.js（轻量 ~60KB）/ ECharts（功能全 ~300KB）

### 7.4 逐题结果补列
`_eval_one` 存 `out["item_id"]`；前端 `resultCols` 增加"题号""垂域"两列。

### 7.5 实施分期
| 期 | 内容 | 依赖 |
|---|---|---|
| **一期（已确认）** | 维度问题分布(带题号,阈值可配默认≤2) + 逐题加题号/垂域列 + 移除类目覆盖列 + ECharts 可视化(柱状图维度问题分布 + 饼图垂域/correctness 分布) | 现有数据，无需改裁判 |
| **二期** | 二级维度问题分布 | 需保留裁判嵌套输出 |

### 7.6 决策结果（已确认）
1. **维度问题阈值**：可配置，默认 ≤2
2. **类目覆盖列**：移除
3. **图表库**：ECharts
4. **二级维度分布**：留二期
